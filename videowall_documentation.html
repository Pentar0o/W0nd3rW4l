Introduction

W0nd3rW4ll est un système complet de gestion de mur d'images permettant de contrôler l'affichage de flux vidéo provenant de caméras IP sur plusieurs écrans. Le système est conçu pour être flexible, scalable et facile à utiliser.

Cas d'usage principaux

  • Centres de surveillance et de sécurité
  • Salles de contrôle industrielles
  • Centres de commandement
  • Affichage d'informations en temps réel
  • Surveillance multi-sites
Note : Ce système est optimisé pour les caméras Axis mais peut être adapté pour d'autres marques grâce aux templates RTSP configurables.

Fonctionnalités

🎥 Gestion multi-caméras

Support de multiples caméras IP avec configuration flexible des flux RTSP et adaptation automatique de la qualité selon le layout.

🖥️ Affichage multi-écrans

Gestion d'un nombre illimité d'écrans avec layouts configurables (1x1, 2x2, 3x3) et détection automatique des murs d'images.

📍 Interface drag & drop

Interface intuitive permettant de glisser-déposer les caméras sur les écrans et d'organiser visuellement la disposition des écrans.

💾 Gestion des scènes

Sauvegarde et restauration de configurations complètes incluant les assignations de caméras et les positions d'écrans.

🔄 Mises à jour en temps réel

Communication WebSocket bidirectionnelle pour des changements instantanés sur tous les écrans connectés.

🎯 Détection de murs d'images

Identification automatique des groupes d'écrans formant un mur d'images 2x2 avec synchronisation des quadrants.

Installation

Installation automatique

Le script d'installation détecte automatiquement votre système d'exploitation et installe les dépendances nécessaires.

chmod +x install_script.sh
./install_script.sh
Le script d'installation effectue les actions suivantes :
  • Détection du système d'exploitation
  • Installation de Python 3.11+ (si nécessaire)
  • Installation des dépendances système
  • Installation des packages Python requis
  • Création de la structure de répertoires
  • Configuration des permissions

Installation manuelle

Si vous préférez installer manuellement :

1. Installer les dépendances système

Sur Linux/Raspberry Pi :

sudo apt-get update
sudo apt-get install python3 python3-pip git curl

Sur macOS :

brew install python@3.11 git curl

2. Installer les dépendances Python

pip3 install -r requirements.txt --break-system-packages

3. Créer les répertoires

mkdir -p cameras scenes templates logs

Configuration

Configuration des caméras

Le fichier cameras/cameras.json contient la configuration de toutes les caméras. Voici la structure :

{
    "cameras": [
        {
            "id": 1,
            "name": "Entrée principale",
            "brand": "Axis",
            "model": "P3265-LVE",
            "ip": "192.168.1.100",
            "login": "root",
            "password": "password",
            "rtsp_template": "rtsp://{login}:{password}@{ip}/axis-media/media.amp?resolution={resolution}",
            "stream_resolution": "640x360",
            "stream_fps": 15,
            "supported_resolutions": ["1920x1080", "1280x720", "960x540", "640x360"],
            "zone": "Extérieur",
            "gps": {
                "latitude": 48.8566,
                "longitude": 2.3522
            }
        }
    ]
}

Templates RTSP

Le système utilise des templates RTSP flexibles pour supporter différentes marques de caméras :

Marque Template RTSP
Axis rtsp://{login}:{password}@{ip}/axis-media/media.amp?resolution={resolution}
Hikvision rtsp://{login}:{password}@{ip}:554/Streaming/Channels/{channel}01
Dahua rtsp://{login}:{password}@{ip}:554/cam/realmonitor?channel={channel}&subtype={stream}
Generic ONVIF rtsp://{login}:{password}@{ip}:554/stream{stream}

Mise à jour automatique des résolutions

Pour mettre à jour automatiquement les résolutions supportées par vos caméras :

./update_resolutions.sh
Important : Cette commande interroge directement chaque caméra pour récupérer ses résolutions 16:9 supportées. Assurez-vous que les caméras sont accessibles sur le réseau.

Utilisation

Démarrage du serveur

./start_server.sh

Le serveur démarre sur le port 1982. L'interface web est accessible à :

URL : http://localhost:1982 ou http://[ADRESSE_IP]:1982

Interface de contrôle

L'interface est divisée en trois zones principales :

1. Barre latérale gauche

  • Scènes sauvegardées : Gestion des configurations
  • Caméras disponibles : Liste des caméras groupées par zone
  • Écrans connectés : Liste des écrans actifs

2. Zone de travail centrale

  • Visualisation de la disposition des écrans
  • Glisser-déposer pour organiser les écrans
  • Indicateurs visuels pour les murs d'images

3. Contrôles

  • Verrouillage : Empêche les modifications accidentelles
  • Arrangement auto : Dispose automatiquement les écrans
  • Réinitialiser : Remet les positions par défaut

Workflow typique

Connecter les écrans
→
Disposer les écrans
→
Assigner les caméras
→
Sauvegarder la scène

Composants principaux

1. Serveur Central (W0nd3rW4ll_Server_Web.py)

  • Serveur Flask pour l'API REST
  • WebSocket via Socket.IO pour la communication temps réel
  • Gestion des configurations et des scènes
  • Détection automatique des murs d'images

2. Interface Web (control.html)

  • Interface utilisateur responsive
  • Gestion drag & drop
  • Communication WebSocket avec le serveur
  • Sauvegarde locale des positions

3. Gestionnaire de résolutions (update_camera_resolutions.py)

  • Interrogation automatique des caméras
  • Détection des résolutions 16:9
  • Mise à jour du fichier de configuration

API REST

Endpoints disponibles

Méthode Endpoint Description
GET /api/cameras Liste toutes les caméras configurées
GET /api/screens Liste tous les écrans connectés
POST /api/screen/{id}/config Met à jour la configuration d'un écran
GET /api/rtsp/{camera_id} Obtient l'URL RTSP d'une caméra
GET /api/scenes Liste toutes les scènes
POST /api/scenes Crée une nouvelle scène
POST /api/scenes/{id}/apply Applique une scène

Exemples d'utilisation

Obtenir la liste des caméras

curl http://localhost:1982/api/cameras

Mettre à jour la configuration d'un écran

curl -X POST http://localhost:1982/api/screen/abc123/config \
  -H "Content-Type: application/json" \
  -d '{
    "layout": "2x2",
    "cameras": [1, 2, 3, 4]
  }'

Dépannage

Problèmes courants

Le serveur ne démarre pas

  • Vérifier que le port 1982 n'est pas déjà utilisé
  • Vérifier les permissions sur les fichiers Python
  • Consulter les logs dans le répertoire logs/

Les caméras ne sont pas accessibles

  • Vérifier la connectivité réseau avec ping [IP_CAMERA]
  • Vérifier les identifiants de connexion
  • Tester l'URL RTSP avec VLC : vlc rtsp://...

Les écrans ne se connectent pas

  • Vérifier que le client peut accéder au serveur
  • Vérifier les règles de pare-feu
  • S'assurer que WebSocket n'est pas bloqué

Logs et débogage

Pour activer les logs détaillés, modifier LOG_LEVEL dans le serveur :

CONFIG = {
    'LOG_LEVEL': logging.DEBUG,
    ...
}

Développement

Ajouter le support d'une nouvelle marque de caméra

Pour ajouter une nouvelle marque, il suffit d'ajouter le template RTSP approprié dans la configuration de la caméra :

{
    "id": 10,
    "name": "Nouvelle caméra",
    "brand": "NouvelleMarque",
    "rtsp_template": "rtsp://{login}:{password}@{ip}:{port}/stream{stream}",
    "login": "admin",
    "password": "password",
    "ip": "192.168.1.200",
    "port": 554,
    "stream": 1
}

Personnaliser les layouts

Pour ajouter de nouveaux layouts (ex: 4x4), modifier :

  1. Dans control.html : Ajouter l'option dans le select
  2. Dans getLayoutDimensions() : Ajouter les dimensions
  3. Dans RTSPManager : Adapter la logique de sélection de résolution

Structure des événements WebSocket

Client → Serveur

  • register_screen : Enregistrement d'un nouvel écran
  • heartbeat : Maintien de la connexion
  • disconnect : Déconnexion d'un écran

Serveur → Client

  • screens_updated : Liste mise à jour des écrans
  • config_update : Nouvelle configuration pour un écran
